## Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub to create your own copy of the repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/supago.git
cd supago
```

### 3. Create a Branch

Create a new branch for your work:

```bash
git checkout -b feat/your-feature-name
```

Recommended branch naming:
- `feat/...`
- `fix/...`
- `docs/...`

---

## Making Changes

While working on your changes:
- Follow the existing project structure
- Keep code simple and readable
- Avoid unrelated changes
- Add comments when logic is not obvious

If you are unsure about something, open an issue first.

---

## Commit Guidelines

Write clear and meaningful commit messages.

Examples:
```text
[Fix][Modules]: handle missing config file
[Feat][Modules]: support table schema command
[Docs]: improve installation guide
```

Avoid messages like:
```text
update
fix bug
changes
```

---

## Pushing Changes

After committing:

```bash
git push origin feature/your-feature-name
```

---

## Opening a Pull Request

1. Open a Pull Request targeting the `main` branch
2. Clearly explain:
   - What was changed
   - Why it was changed
3. Link related issues if available
4. Be open to feedback and revisions

Reviews are meant to improve the project, not attack contributors.

---

## Reporting Bugs

If you find a bug:
1. Check existing issues first
2. Open a new issue with:
   - A clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Logs or screenshots if available

Clear bug reports help a lot.

---

## Feature Requests

Have an idea or improvement?
- Open an issue
- Explain the problem and proposed solution
- Keep it simple and focused

Discussion before implementation is encouraged.
